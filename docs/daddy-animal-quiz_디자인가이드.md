# 🎨 디자인 및 기술 가이드: 아빠와 동물 10고개

본 문서는 '아빠와 동물 10고개' 모바일 웹앱의 시각적 완성도와 원활한 개발을 위한 UX/UI 디자인 가이드 및 기술 스택 제안서입니다.

## 1. 디자인 컨셉 및 톤앤매너 (Tone & Manner)
- **컨셉**: "Playful Safari & Cozy Friends" (즐거운 사파리와 따뜻한 친구들)
- **톤앤매너**: 영유아의 시각적 즐거움과 아빠의 조작 편의성을 모두 만족시킵니다. 날카로운 모서리 없이 둥글고 부드러운 형태(Rounded Shapes), 부드러운 그림자, 친근한 파스텔 톤을 사용하여 현대적이면서도 따뜻한 분위기를 연출합니다.

## 2. 컬러 시스템 (Color Palette)
- 🟡 **Primary (주조색 / 메인 액션)**: `#FFB020` (Sunflower Yellow) - 눈에 띄고 에너지를 주는 노란색. [정답 확인] 등 메인 버튼에 사용합니다.
- 🟢 **Secondary (보조색 / 긍정적 상태)**: `#2EC4B6` (Playful Mint) - 산뜻한 느낌으로 프로그레스 바(진척도)나 보조 긍정 요소에 사용합니다.
- 🔴 **Accent (강조색 / 오답 및 전환)**: `#FF6B6B` (Soft Coral) - 거부감 없는 따뜻한 코럴색. 오답 알림이나 [모르겠어요] 버튼에 사용합니다.
- ⚪ **Background (배경)**: `#F8FAFA` (Light Warm Gray) - 눈이 편안한 밝은 웜그레이. 최상단에 `#FFF9E6` 색상이 은은하게 깔리는 그라데이션을 추천합니다.
- 🌑 **Text (텍스트)**:
  - **메인 텍스트**: `#2D3436` (Dark Charcoal) - 아빠가 힌트를 쉽게 읽을 수 있도록 대비가 높은 진한 회색
  - **서브 텍스트**: `#636E72` (Muted Gray) - 연기 가이드 등 보조 역할 텍스트

## 3. 타이포그래피 (Typography)
앱 내의 **모든 텍스트는 한글**을 기본으로 하며, 가독성과 친근함을 고려하여 다음 폰트를 사용합니다.
- **기본/본문 폰트**: `Pretendard` (또는 `NanumSquareRound`) - 아빠가 흔들리는 화면에서도 힌트를 잘 읽을 수 있도록 가독성이 뛰어난 고딕 기반 폰트
- **타이틀/포인트 폰트**: `BM Jua` (배달의민족 주아체) - 아이들 감성에 맞는 귀여운 디스플레이 폰트
- **크기 체계**:
  - 메인 힌트: `24px ~ 28px` (ExtraBold)
  - 연기 가이드: `14px ~ 16px` (Medium)
  - 버튼 및 입력창: `18px ~ 20px` (Bold)

## 4. 핵심 UI 컴포넌트
- **버튼 (Buttons)**: 터치 영역이 넓은 알약(Pill) 형태 (`border-radius: 9999px`, 최소 높이 `56px`). 터치 시 살짝 줄어드는 반응형 애니메이션 적용.
- **카드 (Cards)**: 배경 `rgba(255, 255, 255, 0.8)`에 모서리 반경 `24px`, 부드러운 그림자(`box-shadow: 0 8px 24px rgba(0,0,0,0.06)`) 적용.
- **프로그레스 바 (Progress Bar)**: 화면 상단에 두께 `8px`로 둥글게 배치하여 직관적인 진척도 표시.
- **결과 모달 (ResultModal)**: 정답 시 노출되는 모달은 동물을 확실히 인지할 수 있도록 이미지 비율을 1:1(`aspect-square`)로 키우되, 좁은 화면에서도 넘치지 않도록 최대 높이(`max-h-[50vh]`)를 제한합니다.

## 5. 모바일 레이아웃 규칙 (Thumb Zone 최적화)
- **최대 너비**: `480px` (모바일 퍼스트, PC/태블릿에서는 중앙 정렬)
- **상단 70% (콘텐츠 영역)**: 힌트 카드가 역순으로 위에서부터 쌓이며, 최신 힌트가 가장 잘 보이도록 상단 고정(스크롤)됩니다.
- **하단 30% (컨트롤 영역)**: 화면 최하단에 어떤 스크롤 환경에서도 항상 고정(Sticky). 긴 텍스트의 보조 버튼 등이 화면 밖으로 밀려 잘리거나 찌그러지지 않도록 레이아웃 방어(`shrink-0`, `min-w-0` 및 줄바꿈 허용)가 엄격하게 적용됩니다.

## 6. 개발 라이브러리 및 기술 스택 (Technical Stack)
GitHub Pages에 정적 사이트로 배포하면서, 프리미엄 UI와 화려한 애니메이션을 효율적으로 구현하기 위해 다음과 같이 기술 스택을 제한 및 확정합니다.

1. **프레임워크: React + Vite**
   - 힌트 상태, 10고개 로직, 화면 전환 등을 컴포넌트 단위로 쉽게 관리할 수 있습니다.
   - Vite를 사용해 매우 빠르게 빌드하고, GitHub Pages에 정적(Static) 배포하기 최적화되어 있습니다.
2. **스타일링: Tailwind CSS**
   - 위에서 정의한 컬러와 컴포넌트 스타일을 가장 빠르고 일관되게 적용할 수 있는 유틸리티 CSS 프레임워크입니다.
3. **애니메이션: Framer Motion**
   - 힌트 카드가 통통 튀며 등장하거나(Bounce), 정답/오답 시 부드러운 전환 효과 등 'Playful'한 분위기를 내기 위한 핵심 라이브러리입니다.
4. **파티클 효과: canvas-confetti**
   - 아이가 정답을 맞혔을 때 화면 전체에 빵 터지는 화려한 꽃가루(Confetti) 효과를 구현하기 위한 가벼운 라이브러리입니다.
