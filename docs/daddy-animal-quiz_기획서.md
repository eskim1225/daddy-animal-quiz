# 🐾 서비스 기획서: 아빠와 동물 10고개

## 1. 서비스 개요
- **서비스명 (가칭)**: 아빠와 동물 10고개
- **타겟 유저**: 5~7세 유아 및 부모 (아빠가 진행자 역할을 하여 힌트를 읽고 연기하며, 아이가 맞추는 형태)
- **서비스 목적**: 아이들의 상상력과 동물 지식 향상, 부모와의 즐거운 오프라인 유대감 형성
- **주요 특징**: 스무고개를 변형하여 한 동물마다 최대 10개의 힌트가 제공되는 **10고개 퀴즈**
- **접근 방식**: 별도의 앱 다운로드/설치 없이, 브라우저 URL을 통해 스마트폰으로 바로 접속 (GitHub Pages 활용)

## 2. 주요 기능 및 화면 흐름

### 2.1. 문제 진행 화면 (모바일 반응형)
- **힌트 시스템 (10고개)**:
  - 한 동물에 대해 10개의 힌트가 준비되어 있으며 처음부터 모두 보여주지 않습니다.
  - [다음 힌트 보기] 버튼을 누를 때마다 1개씩 화면 아래로 누적되어 표시됩니다.
  - **아빠 연기 유도 (Acting Guide)**: 힌트 중 일부는 의성어/의태어와 함께 "(아빠가 크게 소리내주세요)" 같은 가이드가 포함되어 오프라인 상호작용을 극대화합니다.
  - **시각적 힌트 (선택)**: 5~6번째 힌트 단계에서는 텍스트 외에 동물의 실루엣이나 특징적인 부분(꼬리, 발자국) 이미지가 살짝 노출됩니다.
  - **문제 출제 순서**: 앱 접속 시 전체 동물을 무작위로 섞어 매번 다른 순서로 출제하며, 모든 동물이 소진되거나 앱을 새로고침하기 전까지는 한 번 나온 동물이 중복 출제되지 않습니다.
- **아빠의 역할 및 텍스트 UI**:
  - 화면에 표시된 힌트를 아빠가 스마트폰으로 흘깃 보고도 바로 읽을 수 있도록 **매우 큼직하고 굵은 폰트**를 사용합니다.
- **정답 입력 및 컨트롤 영역 (Thumb Zone)**:
  - 아빠가 한 손으로 폰을 들고 조작하기 편하도록 모든 버튼과 입력창은 화면 최하단(엄지손가락 영역)에 고정 배치합니다.
  - 아이가 동물을 외치면, 아빠가 하단 입력창에 동물 이름을 텍스트로 타이핑하고 [정답 확인] 버튼을 누릅니다.
- **모르겠어요 (다음 문제) 버튼**:
  - 아이가 문제를 풀다 포기하고 이 버튼을 누르면, 아쉬움이 남지 않도록 **"정답은 OOO였어요!"라며 정답과 이미지를 화면에 한 번 보여준 후** 다음 동물로 넘어갈 수 있도록 처리합니다.

### 2.2. 정답 및 오답 처리 로직
- **정답 시**: 
  - 화면에 화려하고 귀여운 축하 애니메이션(꽃가루 등)과 함께 "정답입니다!" 문구 출력.
  - 정적 데이터에 미리 세팅해 둔 고화질 무료 이미지(예: Unsplash)를 화면 중앙에 크게 보여줍니다.
  - [다음 동물 만나러 가기] 버튼을 노출하여 다음 문제로 유도합니다.
- **오답 시 및 유연한 정답 인정**:
  - "앗, 아쉽게도 틀렸어요. 다른 동물을 생각해볼까요?" 라는 안내 문구를 부드럽게 띄웁니다.
  - 입력창을 비워주고 계속 기회를 줍니다.
  - **아빠 재량 정답 인정**: 아이가 '강아지'를 '멍멍이'라고 하는 등 정확한 명칭이 아니더라도 맥락상 맞췄다면, 아빠가 직접 통과시킬 수 있는 **[우리아이 정답 인정!]** 보조 버튼을 배치하여 흐름이 끊기지 않도록 합니다.

## 3. 기술 스택 및 데이터 관리
- **프론트엔드 (UI)**: React, Vite, Tailwind CSS, Framer Motion, canvas-confetti
  - **React + Vite**: 컴포넌트 기반 UI 상태 관리(힌트 진행도, 정답/오답 상태 등)와 빠르고 가벼운 정적 사이트 빌드
  - **Tailwind CSS**: 확정된 디자인 가이드(컬러, 둥근 모서리 등)에 맞춘 빠르고 일관성 있는 모바일 퍼스트 반응형 스타일링
  - **Framer Motion & canvas-confetti**: 아이들 눈높이에 맞춘 부드럽고 통통 튀는 애니메이션과 정답 시 화려한 폭죽 효과 구현
- **데이터 저장소 (DB 서버 없음)**: 
  - 앱 내부의 파일(예: `data.js` 또는 `data.json`)에 정적 객체 형태로 동물의 정답, 유의어(옵션), 10개의 힌트 배열, 고화질 이미지 URL을 보관합니다.
- **호스팅 및 배포**:
  - **GitHub Pages**를 활용하여 React(Vite) 빌드 결과물을 정적 사이트로 배포, 비용 없이 도메인 URL만으로 즉시 접속 가능하게 구성합니다.

## 4. 향후 개발 로드맵 (구현 단계)
1. **프로젝트 환경 세팅**: Vite를 활용한 React 프로젝트 뼈대 생성 및 Tailwind CSS, Framer Motion 등 핵심 라이브러리 설치.
2. **모바일 반응형 UI 퍼블리싱**: 아이들 감성에 맞는 부드러운 파스텔톤 컬러와 큼직한 UI 요소, 한 손 조작(Thumb Zone)에 최적화된 레이아웃 적용.
3. **핵심 게임 로직 개발**: 힌트 1개씩 순차 노출, 정답/오답 검증, 아빠 재량 통과 로직, 오답 시 재도전 로직 구현.
4. **콘텐츠 수집 및 채우기**: **40~50종**의 풍성하고 다양한 동물 데이터(동물별 10개 힌트 + 연기 가이드 + 이미지 링크)를 수집하여 `data.js`에 세팅.
5. **테스트 및 GitHub Pages 연동**: 데스크탑/모바일 브라우저에서 최종 테스트 후 GitHub에 올려 URL 발급.
